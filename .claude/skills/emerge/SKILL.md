# Emerge — Find What's Ready to Become Real

Identify idea clusters in the vault that have enough mass to become something concrete.

## Process

1. **Map idea density** — find topics/themes with 3+ notes across different folders

2. **Assess cluster readiness** — for each cluster, check:
   - How many notes? (volume)
   - How spread across time? (persistence)
   - Does it have a clear "why it matters"? (motivation)
   - Is there a natural next action? (actionability)
   - Does it connect to King's active areas or skills? (relevance)

3. **Read the cluster notes** — understand the actual content, not just the topic name

4. **Classify each cluster**:
   - **Ready** — enough material, clear direction, natural next step
   - **Almost** — solid core, missing one piece (research, decision, collaborator)
   - **Brewing** — recurring but still forming, needs more time

5. **Suggest the form** — project, knowledge essay, side project, pitch, experiment

## Output Format

---

**Emerge Report — [date]**

**Ready to become something**

**[Cluster name]**
Type: [project / essay / experiment / product]
Evidence: [[note]], [[note]], [[note]]
Core idea: [1 sentence]
Natural next step: [specific first action]

---

**Almost there**

**[Cluster name]**
What's there: [summary]
What's missing: [the gap]
When it might be ready: [condition or timeframe]

---

**Still brewing**

- [theme] — [N notes, first seen: date]

---

## Rules

- A cluster needs notes across at least 2 different folders to count as emerging
- "Natural next step" must be concrete — not "think more about it"
- Don't force emergence — if nothing is ready, say so
- Cross-reference with `~/Documents/Obsidian/personalOS/Projects/` — don't suggest what already exists
